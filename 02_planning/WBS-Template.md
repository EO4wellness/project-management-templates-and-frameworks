# Work Breakdown Structure (WBS) Template
*A deliverable‑oriented decomposition of the current project work*

---

## Overview
A **Work Breakdown Structure (WBS)** is a hierarchical, deliverable‑oriented breakdown of a project into smaller, manageable components. It organizes milestones, tasks, and subtasks in the order they must be completed, providing a visual and structural foundation for planning, scheduling, estimating, and assigning responsibilities.

A strong WBS helps project teams:
- Visualize the entire project scope  
- Break large work into manageable chunks  
- Estimate costs and timelines  
- Assign roles and responsibilities  
- Track progress and dependencies  
- Prepare for import into tools like Asana, ClickUp, or Jira  

---

## How to Use This Template
Use this template to create a WBS for any project. Follow the three-step decomposition process:

1. **Start with the high-level project picture**  
   Identify major deliverables and milestones.

2. **Break each milestone into tasks**  
   Each task should begin with a **verb** (e.g., “Research venues,” “Draft agenda”).

3. **Break tasks into subtasks**  
   Continue decomposing until each work package is small enough to estimate, assign, and track.

Once complete, you can:
- Convert this WBS into a spreadsheet  
- Import it into Asana using CSV  
- Use it to build your schedule, RACI, communication plan, and risk register

### 6. Preparing for Asana Import 
(or other work management software) [Asana](https://help.asana.com/s/article/your-asana-account?language=en_US)  
When converting your WBS into a spreadsheet:

- Each row = one task or subtask  
- Best practice: start each task with a **verb**  
- Include columns for:
  - Task Name  
  - Description  
  - Assignee  
  - Due Date  
  - Dependencies  
  - Section / Milestone  
- Save as **CSV**  
- Import using:
  - Quick Add → Project → Import CSV  
  - Or Create Project → Import Spreadsheet  

**Note:** CSV import *adds* tasks into Asana; it does not update existing ones.
