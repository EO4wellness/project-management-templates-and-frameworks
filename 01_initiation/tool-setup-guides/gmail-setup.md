# Gmail Setup Guide for Project Initiation

This guide is useful during project initiation when used to configure Gmail to best utilize its features. 

---

## ✔ 1. Configure Project‑Specific Signatures
Create a distinct signature for this project to maintain clarity and include relevant links or contact details.

**How to set it up:**  
Settings (gear icon) → See all settings → General → Signature → Create new → Double check spelling/grammar → Save

**How to use it:**  
When composing an email, click the **Insert signature** icon (pen tool) to switch between signatures.

---

## ✔ 2. Enable & Build Email Templates
Save time on recurring communications such as meeting agendas, weekly updates, onboarding instructions, project newsletters, or stakeholder summaries.

**Enable templates:**  
Settings → See all settings → Advanced → Templates → Enable → Save Changes

**Create a template:**  
Compose → Write your message → More options (three dots) → Templates → Save draft as template → Save as new template

**Use a template:**  
Compose → More options → Templates → Select your template

---

## ✔ 3. Set Communication Norms with the Vacation Responder
Use the Out‑of‑Office tool to manage expectations during high‑intensity phases, off‑site work, or limited availability.

**Set it up:**  
Settings → See all settings → General → Vacation responder

**Configuration tips:**  
- Turn on responder  
- Set start/end dates  
- Add a clear subject and message  
- Optional: Limit responses to “people in my organization”

---

## ✔ 4. Establish Automatic Forwarding & Filters
Ensure critical project emails reach the right people or systems without manual intervention.

**Create a filter:**  
Settings → See all settings → Filters and Blocked Addresses → Create a new filter

**Project Specific Filter Examples:**  
- From: `billing@vendor.com`  
- Subject contains: `[Project Alpha]`  
- Has attachment: true  

**Actions:**  
- Forward to a project alias or distribution list  
- Apply a label  
- Mark as important  
- Skip the inbox (optional)

---

## ✔ 5. Automate Sorting with Labels
Organize project communication from day one.

**Create labels:**  
Left sidebar → Labels → **+** → Create new label  
Example: `Active Projects / Project Alpha`

**Pair with filters:**  
Use filters to automatically apply labels to incoming project emails.

---

## ✔ 6. Enable Undo Send (Critical for PMs)
Prevent accidental or premature sends — especially important for stakeholder communication.

**Set it up:**  
Settings → See all settings → General → Undo Send → Set to **30 seconds**

NOTE: The default undo setting is 5 seconds. 
---

## ✔ 7. Adjust Desktop Notifications (Optional but helpful)
Stay aware of approvals, blockers, or urgent updates without being overwhelmed.

**Set it up:**  
Settings → See all settings → General → Desktop Notifications

**Recommended:**  
- “Important mail notifications only” to reduce noise

**Helpful Tip**
It is good to know where this option is located as you may need to turn it on/off such as when recording screen shots or recordings for training. 

---

## ✔ 8. Optional: Create a Project Alias (If Allowed)
Some organizations allow secondary “send as” addresses.

**Why use it:**  
- Separates project communication  
- Improves filtering  
- Provides clarity for stakeholders  

**Set it up:**  
Settings → See all settings → Accounts → Send mail as → Add another email address

---

## ✔ 9. Quick Self‑Check (PM Thinking Prompts)
Use these questions to ensure your Gmail setup supports your project’s communication needs:

- Have I created signatures and templates that reduce repetitive work?  
- Do filters and labels route project emails cleanly?  
- Have I set expectations for availability or response times?  
- Does this setup support asynchronous contributors?  
- Are critical emails automatically forwarded or highlighted?  
- Is my inbox structured to prevent missed approvals or blockers?  
- Have I integrated needed tools such as tasks, calendar, zoom or other meeting software? 

---

## Final Review
Your Gmail environment should now support:

- clear, consistent communication  
- reduced inbox noise  
- predictable workflows  
- inclusive participation (async + sync)  
- alignment with your Communication Plan  

This setup prevents administrative bottlenecks and supports smooth execution throughout the project lifecycle.
