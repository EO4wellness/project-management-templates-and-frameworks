# Tool Setup Guides (Project Initiation)

This folder contains lightweight, practical one‑page guides to help configure during our project intiation phase some of the more common Project Management tools. These guides are not software manuals — they are project‑focused setup checklists and thought prompts designed to support clarity, communication, and smooth collaboration from day one.

Each guide is intentionally short, actionable, and ecosystem‑agnostic where possible. The goal is to help project managers quickly establish a clean, organized working environment that aligns with the project’s communication plan, stakeholder needs, and deliverables.

---

## Common Tool Types 

    Communication: How we interact. (Gmail, Slack, Teams)

    Productivity: How we create. (Google Docs, Excel, Notion)

    Scheduling and Work Management: How we align on tasks and track progress. (GitHub Projects, Jira, Asana, Trello)

## Philosophy

This folder reflects the principle that **strong initiation prevents chaos later**.  
Setting up your tools intentionally — even at a basic level — supports:

- clear communication  
- predictable scheduling  
- organized documentation  
- accessible collaboration  
- inclusive participation (async + sync)  
- reduced noise and fewer bottlenecks  

These guides help you think through *how* your tools support your project, not just *how to click buttons*.

---

## How to Use These Guides

1. Start with the **Generic Tool Setup Guide** to clarify your needs.  
2. Use the tool‑specific one‑pagers to configure your environment.  
3. Adjust based on your project’s deliverables, team preferences, and communication plan.  
4. Add new guides or expand existing ones as your toolkit grows


This folder is intentionally evolving — add what you need, when you need it.
